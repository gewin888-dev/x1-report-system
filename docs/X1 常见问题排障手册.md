# X1 常见问题排障手册

更新时间：2026-05-22

---

## 1. 服务无法启动

**现象**：执行启动脚本后服务未运行。

**排查步骤**：
1. 检查端口占用：`lsof -nP -iTCP:8082 -sTCP:LISTEN`
2. 检查 PID 文件是否失真：`cat x1.pid`
3. 查看应用日志：`tail -50 logs_x1/app_$(date +%Y%m%d).log`
4. 使用标准诊断：`bash doctor_x1_daemon.sh`
5. 必要时重新拉起：`bash restart_x1_daemon.sh`

**常见原因**：
- 8082 端口被占用
- 依赖缺失
- 配置文件损坏
- 启动后导入阶段报错

---

## 2. 端口被占用

```bash
lsof -nP -iTCP:8082 -sTCP:LISTEN
kill <PID>
bash start_x1_daemon.sh
```

如不确定现场状态，优先：
```bash
bash stop_x1_daemon.sh
bash start_x1_daemon.sh
```

---

## 3. 模板缺失 / 导出失败

**现象**：导出提示模板不存在、导出结构空、模板命中异常。

**排查步骤**：
1. 确认模板根目录存在
2. 检查 `x1_config.json` 中模板相关路径
3. 后台 → 模板管理，检查对应对象模板是否有效
4. 检查模板注册与对象映射是否一致

---

## 4. 数据库锁定 / 写入失败

**现象**：报错 `database is locked` 或写入失败。

**处理建议**：
1. 先确认是否有多个进程同时操作
2. 优先使用标准重启：
   ```bash
   bash restart_x1_daemon.sh
   ```
3. 若问题反复出现，再检查是否有异常脚本或僵尸进程持续占库

说明：X1 为双 SQLite 结构，检查问题时要区分：
- 根库 `x1_data.db`
- 业务库 `data/x1_data.db`

---

## 5. 页面加载异常 / JS 修改不生效

这是 X1 当前非常重要的高频误判点。

### 员工端
当前员工端运行页真实入口是：
- `templates/record_index.html`
- `static/record.js`

不要默认认为修改 `static/src/record/*` 或重新构建 `static/dist/record.bundle.js` 就会立刻生效。

### 管理端
当前管理端核心运行脚本是：
- `static/admin.js`

### 排查步骤
1. 强制刷新浏览器缓存（Cmd+Shift+R）
2. 核对模板实际加载的 JS 文件
3. 修改后如涉及运行态变更，重启服务：
   ```bash
   bash restart_x1_daemon.sh
   ```
4. 再重新打开页面验证

---

## 6. 权限修改后看起来没生效

**排查顺序**：
1. 先看 `role_permission_final` 是否已正确反映最终权限
2. 再看当前页面是否刷新了权限显示
3. 再看该用户是否仍在旧会话中
4. 必要时让目标用户重新登录

注意：
- `role_permissions` 是旧遗留表，不应再当当前权限真相看

---

## 7. 任务流程显示或操作异常

当前员工端主业务口径应为：
- 待检测
- 检测中
- 检测完成

员工端当前只保留：
- 进入录入
- 完成任务

如果页面仍出现：
- 接单
- 开始执行
- 已接单
- 执行中

优先怀疑：
1. 页面仍在跑旧脚本
2. 缓存未刷新
3. 文档理解与运行事实不一致

---

## 8. 健康检查

```bash
curl -s http://127.0.0.1:8082/api/x/health | python3 -m json.tool
bash doctor_x1_daemon.sh
```

如健康检查异常，不要先猜前端问题，先确认服务本身是否正常启动。
