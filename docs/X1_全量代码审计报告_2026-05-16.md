## P0 — 阻断性 / 高危问题

### P0-1 🔴 collectData 缺失 `airchange_speed_only` 和 `pass_box_volume` 采集

**文件**: `static/record.js` collectData()  
**影响对象**: `pass_box`（传递窗）  
**问题**: `collectData()` 有 20 种 inputType 的显式采集分支，但遗漏了 `airchange_speed_only` 和 `pass_box_volume`。这两种类型仅用于传递窗（箱体内尺寸 + B1/B2/B3型换气次数）。

**后果**: 传递窗的箱体内尺寸和换气次数数据在提交/导出时**静默丢失为空**，报告中对应字段为空。  
**验证**: 已确认 renderAirchangeSpeedOnly 使用 `data-dp-airspeedonly` 属性，但 collectData 中仅有 `itype==='airchange'` 分支（匹配 `data-itype="airchange"`），`airchange_speed_only` 的 `data-itype` 不会命中任何分支。

**修复建议**:
```javascript
} else if(itype==='airchange_speed_only'){
    const vents=[];
    pb.querySelectorAll('[data-vent-row]').forEach(row=>{
        const a=row.querySelector('[data-va]')?.value;
        const s=row.querySelector('[data-vs]')?.value;
        if(a||s) vents.push({area:a,speed:s});
    });
    if(vents.length) params[p.key]={type:'airchange_speed_only',vents,result};
} else if(itype==='pass_box_volume'){
    const l=pb.querySelector('[data-dim="length"]')?.value;
    const w=pb.querySelector('[data-dim="width"]')?.value;
    const h=pb.querySelector('[data-dim="height"]')?.value;
    if(l||w||h) params[p.key]={type:'pass_box_volume',length:l,width:w,height:h,result};
}
```

### P0-3 🔴 登录接口无暴力破解防护

**文件**: `app_x1.py` L891-920  
**问题**: `/login` POST 无频率限制、无失败锁定、无验证码。系统监听 `0.0.0.0` 直接暴露网络。  
**后果**: 弱密码账号（最低 6 位）可在分钟级别被暴力破解。

**修复建议**: 
- 实现 IP + 用户名维度失败计数器
- 5 次失败后锁定 15 分钟
- 短期可用 `flask-limiter` 限制 `/login` 每 IP 每分钟 10 次

### P0-5 🔴 客户管理"删除"按钮 onclick 语法错误 — 按钮失效

**文件**: `static/admin_customers.js` L114  
**问题**: 
```html
onclick="event.stopPropagation()" ;deleteCustomer('xxx')"
```
引号在 `)` 后闭合，`;deleteCustomer(...)` 变成裸文本。**删除按钮永远不会触发 deleteCustomer 函数。**

**修复建议**:
```javascript
onclick="event.stopPropagation();deleteCustomer('xxx')"
```

### P0-6 🔴 GMP 车间 A 级模板双路径冲突

**文件**: `template_semantic_mappings.json` + `template_registry.json`  
**问题**: GMP A 级存在两套 key 指向**不同物理文件**：
- 语义默认 `pharma/gmp_workshop/a` → 公司资料正式模板
- Rules 解析 `pharma/gmp_workshop/grade/a` → uploads 测试模板

**后果**: 语义映射开/关时导出使用不同模板版本  
**修复建议**: 统一为一套 key，删除老 key 或将语义默认改为新 key

### P0-7 🔴 眼科手术室模板强依赖语义映射 — fallback 指向错误

**文件**: `template_rules.py` L350-370  
**问题**: 眼科手术室在 `resolve_template_rule()` 中走普通手术室分支，只有语义映射能覆盖到正确的眼科模板。**如果语义映射被删除或损坏，眼科手术室将使用普通手术室模板导出。**

**修复建议**: 在 template_rules 中增加眼科手术室独立分支

### P0-8 🔴 手术室辅房模板 key 新旧体系共存

**文件**: `template_rules.py` TEMPLATE_RULE_REGISTRY + `template_registry.json`  
**问题**: 规则引擎使用旧 key（`aux/level1-local5-surround6`），registry overlay 注册新 key（`aux/level1`）。语义链断裂时 fallback 到 builtin 的"通用洁净功能用房"模板。

**修复建议**: 更新 TEMPLATE_RULE_REGISTRY 使用新 key
