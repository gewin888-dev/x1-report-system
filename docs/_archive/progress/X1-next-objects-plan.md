# X1 第二批对象接入计划（第一版）

## 当前目标

在 `pass_box`、`laminar_hood` 之后，开始为更复杂对象接入 X 做准备。

候选对象：
1. `operating_room`
2. `clean_function_room`
3. `bsl`

---

## 当前判断

### 1. `operating_room`
特点：
- 房间类型与等级联动明显
- 存在“手术室 / 洁净辅房”分支
- 更能检验对象注册机制和工作区扩展能力

### 2. `clean_function_room`
特点：
- 有子房间概念
- 等级和子房间存在映射关系
- 复杂度中等偏上

### 3. `bsl`
特点：
- 上下文维度多
- 生物安全等级 / 洁净等级 / 屏障环境等链路更复杂
- 适合后续作为更高复杂度对象

---

## 当前建议顺序

### 第一候选：`operating_room`
原因：
- 复杂度足够高，能检验 X 结构
- 比 bsl 更适合作为第二阶段对象入口
- 能较快暴露对象注册、工作区、导出适配的真实边界

### 第二候选：`clean_function_room`
### 第三候选：`bsl`

---

## 接入原则

1. 不复制 T 的旧 ready/source/live/saved 逻辑
2. 只提炼对象结构和业务值
3. 页面差异尽量留在对象模板/适配器层
4. 继续坚持统一 canonical model
