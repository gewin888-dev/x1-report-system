# X1 洁净等级语义层（第一版）

## 为什么要有这一层
X 不能把洁净等级只当一个字符串字段处理。

原因：
- 不同领域表达方式不同
- 不同标准体系语义不同
- 有些表达形式相似，但不能直接等同
- 等级不仅影响显示，还影响模板规则、参数规则、报告语义

因此，X 需要逐步建立：
- 领域
- 标准体系
- 等级表达原文
- 对象分支
- 业务语义说明

---

## 第一批先做透的对象
1. `operating_room`
2. `clean_function_room`
3. `bsl`

## operating_room（医院洁净部 / GB 50333-2013）

### 主手术室链
- 标准：`GB 50333-2013`
- 对象：`operating_room`
- 分支：`手术室 / 眼科手术室`
- 等级表达：
  - `Ⅰ级（百级）`
  - `Ⅱ级（千级）`
  - `Ⅲ级（万级）`
  - `Ⅳ级（十万级）`
- 语义说明：
  - 属于医院洁净手术部的主手术室等级表达
  - 不能直接与食品、制药、生物安全、电子等领域的等级表达混同

### 洁净辅房链
- 标准：`GB 50333-2013`
- 对象：`operating_room`
- 分支：`洁净辅房 / 辅房`
- 等级表达：
  - `Ⅰ级（局部5级其他6级）`
  - `Ⅱ级（7级）`
  - `Ⅲ级（8级）`
  - `Ⅳ级（8.5级）`
- 语义说明：
  - 这是洁净辅房链自己的表达，不等于主手术室链
  - 会影响后续参数覆盖规则（如洁净度/细菌浓度表达）

## bsl（生物安全）
- 需要进一步提炼：
  - BSL 等级 vs 洁净等级的区别
  - 生物安全等级不是洁净等级
  - 洁净等级在该对象里通常还要结合 ISO 表达

## 当前结构建议
未来 X 至少要逐步具备这样的等级语义对象：

- `domain`
- `standard_code`
- `object_type`
- `object_branch`
- `level_raw`
- `level_semantic_key`
- `semantic_note`
- `impacts`
