# X1 正式发布路线图（第一版）

## 目标

把 X 从当前“独立骨架 + 样板对象 + 基础导出链”推进到“可正式发布”的系统。

---

## 一、当前已具备的发布基础

### 1. 独立空间
- 独立目录
- 独立端口
- 独立运行态目录
- 独立 API 前缀

### 2. 基础主链
- 页面录入样板
- Draft 预览
- 暂存
- 草稿读取与回填
- JSON 导出
- DOCX 样板导出

### 3. 基础层次
- 配置层
- 启动入口层
- 服务层
- 模型层
- 工作区层
- 报告适配层

---

## 二、距离“正式发布”还差的关键项

### A. 数据与模型层
1. canonical model 继续稳定化
2. 对象注册机制继续实装化
3. 减少控制器里的对象差异散点
4. 形成更明确的 room / export adapter 结构

### B. 对象覆盖层
当前只有：
- pass_box
- laminar_hood

后续至少应继续接入：
- operating_room
- clean_function_room
- bsl

### C. 导出层
1. 从样板 docx 逐步过渡到更正式的模板输出
2. 建立对象级报告 adapter
3. 明确模板选择规则

### D. 启动与交付层
1. 启动入口继续整理
2. 配置继续集中化
3. 资源路径进一步可迁移化
4. 为打包执行程序准备 launcher / resource / icon / packaging 结构

### E. 验收与发布层
1. 建立对象级真实验收清单
2. 建立最小回归测试清单
3. 形成阶段发布版本标记

---

## 三、建议的推进优先级

### 第一优先级
- 继续稳固 current architecture
- 扩对象到 `operating_room` / `clean_function_room` / `bsl`
- 继续做对象注册化

### 第二优先级
- 强化导出适配层
- 让 docx 从样板报告走向正式模板报告

### 第三优先级
- 完善启动与打包准备
- 为正式发布建立交付壳

---

## 四、当前判断

X 已经进入“可以认真推进正式发布路线”的阶段，但距离正式发布仍有明显工程量。

现在最重要的是：
- 不偏离终极目标
- 不回到补丁式开发
- 继续按层次推进
