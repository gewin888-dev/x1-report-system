# X1 启动与封装准备说明

## 当前目标

X1 后续可能封装为可执行程序，因此当前结构开始按“可打包、可迁移、可独立启动”整理。

---

## 当前入口

- 主入口：`app_x1.py`
- 配置文件：`x1_config.json`

### 当前启动方式
```bash
python3 /Users/fuwuqi/检测报告生成系统_X1/app_x1.py
```

访问地址：
- `http://127.0.0.1:8082/`

---

## 为什么增加配置文件

后续如果要封装为执行程序，不能让端口、目录、资源路径全部散落写死在代码里。

因此 X 当前开始建立：
- 独立配置文件
- 清晰入口
- 相对化路径组织

这会为后续：
- 桌面程序封装
- 独立分发
- 启动器封装
- 环境迁移
提供基础。

---

## 后续建议方向

1. 把更多路径配置继续收敛到 `x1_config.json`
2. 逐步形成 `launcher` / `main` 启动入口
3. 为未来打包成可执行程序预留图标、资源、启动器结构
4. 避免主链继续依赖临时测试脚本
