# X1 启动入口说明（执行程序准备）

## 当前新增入口

### Python 启动器
- `launcher_x1.py`

作用：
- 作为 X1 的独立启动入口
- 为未来封装为执行程序时提供统一 main 入口

### Shell 启动脚本
- `start_x1.sh`

作用：
- 便于本地直接启动
- 为未来桌面快捷方式或打包器调用提供基础

---

## 当前建议启动方式

### 方式1：直接运行 shell 启动脚本
```bash
/Users/fuwuqi/检测报告生成系统_X1/start_x1.sh
```

### 方式2：运行 Python 启动器
```bash
python3 /Users/fuwuqi/检测报告生成系统_X1/launcher_x1.py
```

---

## 为什么现在就要做启动器

因为 X 后续可能封装为可执行程序，主入口必须尽量清晰：
- `launcher_x1.py` 可以作为未来打包 main
- `app_x1.py` 保持为服务实现入口
- 避免把“服务实现”与“程序启动方式”混成一层
