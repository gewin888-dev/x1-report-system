# X1 clean_function_room / bsl 规则观察（第一轮）

## 目标
继续推进 X 的等级语义层与模板规则层，但不急着假装已经做完对象；先提炼 T 中这两个对象最容易出错的业务点。

---

## 一、clean_function_room

### 已确认的关键业务点
1. `clean_function_room` 在 T 中先选 `subroom`
2. `subroom` 会进入：`cleanFunctionSubroom`
3. 存在 `subroomStandardMap`
   - `通用洁净功能用房` → 默认 `Ⅲ级（万级）`
   - `ICU病房` → `Ⅲ级（万级）`
   - `消毒供应中心` → `Ⅲ级（万级）`
   - `透析室` → `Ⅲ级（万级）`
4. 随后页面只允许从：
   - `Ⅲ级（万级）`
   - `Ⅳ级（十万级）`
   中选择

### 当前判断
- `clean_function_room` 不是“任意等级 + 任意房间”
- 子房间语义是第一层
- 洁净等级是建立在子房间语义之后的第二层
- 这类对象以后模板规则不能只认 `Ⅲ级/Ⅳ级`，必须同时认子房间

---

## 二、bsl

### 当前已确认的关键点
1. T 页面中 `bsl` 相关 UI 与普通洁净等级分开
2. 生物安全领域额外存在：
   - 生物安全等级入口
   - 洁净等级入口
   - 屏障/上下文类信息
3. 这再次说明：
   - `BSL等级` ≠ `洁净等级`
   - 两者在对象语义上是并列但不同的事实

### 当前判断
- `bsl` 对象以后不能只存一个等级字段
- 至少要保持：
  - 生物安全等级事实
  - 洁净等级事实
  - 可能还有上下文事实
- 模板规则未来也不能只按单等级字符串判断
