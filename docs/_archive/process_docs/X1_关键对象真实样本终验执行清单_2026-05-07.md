## 三、单对象终验检查表

每个对象/场景终验时，统一检查以下 7 项：

1. **场景命中**：业务输入是否命中正确模板
2. **封面字段**：项目名称 / 委托单位 / 报告编号 / 地址 / 联系方式
3. **正文字段**：检测日期 / 检测状态 / 气象条件 / 判定依据 / 检测结论 / 对象特有字段
4. **表格结构**：是否写进正确单元格，是否有 run 分裂、串位、漏填、错列
5. **结论表**：检测结果列是否正确写入，是否与判定一致
6. **文件产物**：命名、目录、可打开性是否符合正式归档预期
7. **结果分级**：终验通过 / 基本通过（带小问题）/ 需整改 / 失败

## 四、当前已验证有效的方法

本轮已基本验证一套可复制做法：
1. **哨兵验证**：先看 semantic / template / resource 主链
2. **逐层修正**：每修一层立即复跑，让更深层问题自然暴露
3. **内容探针**：主链通过后再检查 `.filled.docx` 基础字段写入
4. **结果分层表达**：模板治理通过 ≠ 真实样本终验通过 ≠ 系统整体验收通过

- semantic key 格式未归一型
- 资源文件名旧口径型
- 多场景共享模板型
