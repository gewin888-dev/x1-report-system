## 一、总体结论

当前 X1 项目中，这类文档**总计约 75 个**（含 backups 历史副本）；其中当前 `docs/` 主目录下约 **44 个**。

当前真正的问题不是“缺文档”，而是：

1. **当前活文档与历史施工单混放**
2. **部分旧文件虽然未删除，但已不应再参与当前判断**
3. **少数文件头已经降级，但正文或文件名仍容易误导后续会话 / 后续模型**
4. **缺少统一的文档状态标签体系**

因此，建议把现有文档统一分为四类：

- `ACTIVE`：当前活文档，持续维护
- `REFERENCE`：辅助参考，不承担当前主线/最终验收判断
- `ARCHIVE`：历史过程材料，保留回溯价值，但不再参与当前状态判断
- `OBSOLETE`：已失效/停止维护，仅作历史快照
