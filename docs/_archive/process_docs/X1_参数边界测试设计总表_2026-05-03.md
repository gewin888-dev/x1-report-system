# X1 参数边界测试设计总表（第一版）

更新时间：2026-05-03 08:45

## 目的
把“全参数测试”从主链导出验证中剥离出来，建立专门面向**判定边界**的测试体系。

## 设计原则
1. 每个参数至少覆盖 5 个点：
   - 下限外一点
   - 下限值
   - 下限内一点
   - 上限内一点
   - 上限外一点
2. 每条边界 case 必须留下：
   - 输入参数
   - 预期判定
   - 实际判定
   - judgement_engine
   - judgement_reason
   - abnormal_items
   - 暂存 / 报告 / 飞书留痕
3. 主链记录与边界记录分开管理，避免混淆。
