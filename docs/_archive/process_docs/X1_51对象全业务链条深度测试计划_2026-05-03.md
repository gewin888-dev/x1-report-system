## 测试总原则
每一项深测不只看“能导出”，至少覆盖以下 6 层：

1. **前台录入层**
   - 领域进入
   - 检测类型选择
   - 子类型/等级/上下文链闭合
   - 参数区渲染正常

2. **前台数据链**
   - collectData 正常
   - 保存草稿正常
   - 草稿读取正常
   - 页面回填正常
   - 通用结果字段 / abnormal_items 回填正常

3. **后端规范化层**
   - payload normalize 正常
   - 非法对象 / 缺字段有守门
   - `export_type == room.type_id`

4. **后端判定层**
   - judgement_engine 有输出
   - judgement_reason 有输出
   - judgement_overridden 行为正确
   - abnormal_items / item_results 与异常样本一致

5. **导出层**
   - template_key 命中正确
   - 模板文件存在
   - 生成 docx 成功
   - 关键字段落位可核查

6. **断网可用层**
   - 离线暂存可用
   - 离线记录列表可见
   - 恢复联网后可继续提交/导出
   - 不因断网丢失参数、判定摘要、异常项摘要

### Phase 1：51 项业务对象/组合清单落地
目标：形成可执行清单，而不是停留在概念口径。

输出物：
- `docs/X1_51对象深测执行台账_2026-05-03.md`

每条至少记录：
- 编号
- 对象
- 子类型/等级

- 是否已有真人导出

### Phase 2：在线主链深测
目标：51 项全部跑通在线场景。

每项至少做 3 类样本：
1. **最小合法样本**
2. **标准正常样本**
3. **异常判定样本**

1. electronics_workshop 5级
2. food_workshop 4级
3. veterinary_gmp_workshop 4级
4. 其余 38 项做深度补测（重点补异常样本和页面回填样本）

### Phase 3：断网场景专项
目标：确认断网不只是“能本地保存”，而是业务继续可用。

每项至少覆盖：
1. 在线录入中断网 -> 本地保存
2. 断网新建 -> 本地草稿可见
3. 恢复联网 -> 可继续读取 / 提交 / 导出
4. 断网前已有 summary / abnormal_items -> 不丢

### Phase 4：全链路验收汇总
输出物：
- 51 项总通过率
- 高风险未过项
- 可发布 / 不可发布判断
