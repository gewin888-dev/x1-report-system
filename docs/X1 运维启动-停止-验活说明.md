# X1 运维启动-停止-验活说明

**更新时间**: 2026-05-16

---

## 守护进程脚本

所有脚本位于项目根目录 `/Users/fuwuqi/检测报告生成系统_X1/`：

| 脚本 | 功能 |
|------|------|
| `start_x1_daemon.sh` | 启动服务（含端口检查、PID 管理） |
| `stop_x1_daemon.sh` | 停止服务（优雅停止 → 强制停止） |
| `restart_x1_daemon.sh` | 重启服务（stop + start） |
| `status_x1_daemon.sh` | 查看当前状态 |
| `doctor_x1_daemon.sh` | 完整健康诊断 |

## 快速操作

```bash
cd /Users/fuwuqi/检测报告生成系统_X1

# 启动
bash start_x1_daemon.sh

# 停止
bash stop_x1_daemon.sh

# 重启
bash restart_x1_daemon.sh

# 查看状态
bash status_x1_daemon.sh

# 健康诊断
bash doctor_x1_daemon.sh
```

## 图形化管理台

双击桌面 `XT系统启停管理/X1管理台.command` 可打开终端菜单界面。

## PID 管理

- PID 文件：`x1.pid`（项目根目录）
- 启动时写入，停止时清理
- 如果 PID 文件存在但进程已死，启动脚本会自动清理

## 端口检查

- 服务端口：**8082**
- 启动前自动检查端口占用
- 如果端口被其他进程占用，启动会失败并提示

## 健康接口

```
GET http://127.0.0.1:8082/api/x/health
```

返回 `{"success": true, ...}` 表示服务正常。

## 日志位置

| 日志类型 | 路径 |
|----------|------|
| 应用日志 | `logs_x1/app_YYYYMMDD.log` |
| 守护脚本日志 | `logs/x1_YYYYMMDD.log` |
| 手动重启日志 | `logs_x1/manual_restart_*.log` |

## 自动备份

- LaunchAgent: `~/Library/LaunchAgents/com.x1.backup.plist`
- 执行时间：每天凌晨 2:00
- 备份目录：`/Users/fuwuqi/backups_x1/`
- 保留策略：30 天 / 最多 60 份
