# X1 飞书月目录自动切换运维说明

## 当前规则

X1 现在支持三种飞书目录模式：

1. **direct（直传目录）**
   - `feishu.folder_reports` / `feishu.folder_exports` 直接指向最终上传目录
   - 系统直接上传，不再额外套年份目录

2. **month-root（月目录父级）**
   - 配置 token 指向“月份目录父目录”
   - 系统会按当前月份自动查找或创建 `YYYY-MM` 子目录并上传
   - 这是后续推荐模式

3. **year-root（年份目录父级）**
   - 配置 token 指向“年份目录父目录”
   - 系统会按当前年份查找或创建 `YYYY` 子目录并上传
   - 兼容旧模式，但不适合月维度自动归档

## 推荐做法

### 检测报告
- 将 `feishu.folder_reports` 配成“检测报告总父目录”
- 该目录下按月维护 `YYYY-MM`

### 原始记录
- 将 `feishu.folder_exports` 配成“原始记录总父目录”
- 该目录下按月维护 `YYYY-MM`

这样系统到新月份时会自动命中或创建当月目录，不需要每月手动改 token。

## 每月巡检动作

1. 后台执行一次“测试飞书配置”
2. 检查返回结果中的：
   - `reports_meta.mode`
   - `exports_meta.mode`
   - `warnings`
3. 若提示“当前仍为按年模式”或“未命中本月目录”，按提示处理
4. 再跑一次 `python3 test_feishu_probe.py` 做实测

## 验收标准

以 `test_feishu_probe.py` 为准：

- `report_success = true`
- `export_success = true`
- `SUMMARY PASS`

## 故障经验

若出现：
- `parent node out of sibling num.`

优先怀疑：
- 配置 token 指向了异常目录节点
- 年目录/历史目录可列出但不可作为上传 parent_node

此时优先改用：
- 新鲜可写的月目录
- 或切回月目录父级自动创建策略
