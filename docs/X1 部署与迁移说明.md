# X1 部署与迁移说明

**更新时间**: 2026-05-16

---

## 系统要求

- macOS（推荐 12+）
- Python 3.9+
- Node.js（仅前端构建需要，运行时不需要）
- 磁盘空间：系统本体 ~100MB，数据视业务量而定

## 迁移步骤

### 1. 复制项目目录

```bash
# 整体复制（排除运行时文件）
rsync -av --exclude='logs_x1/' --exclude='cache_x1/' --exclude='temp_x1/' \
  --exclude='.git/' --exclude='node_modules/' \
  源机器:/path/检测报告生成系统_X1/ /Users/新用户/检测报告生成系统_X1/
```

### 2. 复制外部数据

```bash
# 模板文件
rsync -av 源机器:~/公司资料/检测部/检测报告模板/ ~/公司资料/检测部/检测报告模板/

# 归档目录
rsync -av 源机器:~/公司资料/检测部/检测报告/ ~/公司资料/检测部/检测报告/
rsync -av 源机器:~/公司资料/检测部/原始记录/ ~/公司资料/检测部/原始记录/
```

### 3. 更新配置路径

检查 `x1_config.json`：
- `template_base`：确认 `~` 能正确展开到新用户目录
- `archive.formal_report_archive`：更新为新机器绝对路径
- `archive.formal_raw_archive`：更新为新机器绝对路径

### 4. 初始化环境

```bash
cd /Users/新用户/检测报告生成系统_X1
bash init_x1_env.sh      # 创建工作目录 + 补齐配置
bash install_x1.sh       # 安装 Python/Node 依赖
```

### 5. 验证

```bash
bash start_x1_daemon.sh  # 启动服务
python3 smoke_test_x1.py # 冒烟测试
curl http://127.0.0.1:8082/api/x/health  # 健康检查
```

### 6. 配置自动备份

```bash
# 复制 LaunchAgent
cp com.x1.backup.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.x1.backup.plist
```

## 注意事项

- `feishu_config.json` 包含敏感凭证，迁移时注意安全
- 数据库文件 `x1_data.db` 迁移时确保没有进程在写入
- 如果用户名变化（如 gewin → fuwuqi），需要全局检查硬编码路径
- `static/docs_ui.js` 中的文档路径需要更新
