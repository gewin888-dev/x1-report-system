**运行环境**: macOS / Flask / SQLite  
**服务端口**: 8082

---

## 一、系统定位

X1 是内蒙古普迪检测技术服务有限公司当前在用的检测报告生成业务系统，覆盖：

- 检测员录入与导出
- 项目与任务联动
- 客户资料与客户反馈
- 后台管理与权限治理
- 运维、备份恢复与基础排障

它不是单纯的表单导出工具，而是当前实际承载检测录入、项目任务、客户资料与后台治理的一套生产系统。

---

## 二、当前运行事实

### 1. 真实前端入口
- 员工端运行页：`templates/record_index.html`
- 员工端真实脚本：`static/record.js`
- 管理端真实脚本：`static/admin.js`

> 项目中虽然存在历史构建链，但当前运行页并不以 bundle 作为唯一真入口。涉及员工端行为修改时，应优先核对 `record_index.html` 实际加载的脚本。

### 2. 数据库结构
当前系统采用双 SQLite 数据库：

- 根目录 `x1_data.db`
  - 用户
  - 权限
  - 日志
  - 记录索引类数据
- `data/x1_data.db`
  - business_projects
  - project_tasks
  - client_profiles
  - client_feedback
  - report_feedback
  - 其他业务数据

### 3. 权限真相来源
当前权限运行真相以：
- `role_permission_final`

为准。

旧表 `role_permissions` 仅视为历史遗留参考，不再作为当前权限真相来源。

### 4. 当前任务业务口径
员工端当前主流程按三态理解：

- 待检测
- 检测中
- 检测完成

员工端只保留两个主动作：

- 进入录入
- 完成任务

### 5. 当前运维主方式
当前受控启停方式以项目根目录脚本为准：

- `start_x1_daemon.sh`
- `stop_x1_daemon.sh`
- `restart_x1_daemon.sh`
- `status_x1_daemon.sh`
- `doctor_x1_daemon.sh`

## 四、当前维护原则

1. 以真实运行入口为准，不以历史构建链假设为准。
2. 以 `role_permission_final` 为权限真相，不回退到旧补丁表理解。
3. 以当前三态任务业务口径为准，不再把旧员工流程当主流程。
4. 以标准启停脚本作为唯一受控重启入口。
5. 文档若与运行事实冲突，以运行事实为准，并及时修正文档。
