# X1 版本号管理规则

**更新时间**: 2026-05-16

---

## 版本号格式

```
X<主版本>.<次版本>
```

- **X** 前缀：标识 X1 系统（区别于旧 T 系统）
- **主版本**：重大架构变更或里程碑发布时递增
- **次版本**：功能迭代、Bug 修复时递增

当前版本：**X4.8**

## 版本号同步位置

| 位置 | 文件/字段 |
|------|-----------|
| 配置文件 | `x1_config.json` → `version` |
| 健康接口 | `/api/x/health` → `version` |
| 前端显示 | 后台页面底部版本标识 |
| Git tag | 重要发布时打 tag（如 `X4.8`） |

## 升级条件

| 变更类型 | 版本动作 |
|----------|----------|
| 新增检测对象 | 次版本 +1 |
| 新增功能模块 | 次版本 +1 |
| Bug 修复 / UI 调整 | 次版本 +1 |
| 架构重构 / 数据库变更 | 主版本 +1 |
| 迁移部署（无功能变更） | 不变 |

## 备份留档口径

- 每次主版本升级前，执行一次完整备份（`backup_x1.sh`）
- 备份文件名包含版本号和时间戳：`X1_auto_YYYYMMDD_HHMMSS.tar.gz`
- Git commit message 中标注版本号变更
